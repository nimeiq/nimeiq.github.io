Vue组件

为了拆分Vue实例的代码量，用不同的组件，来划分不同的功能模块

  • 组件化：是从代码逻辑的角度进行划分的；方便代码分层开发，保证每个功能模块的职能单一；
  • 模块化：是从UI界面的角度进行划分的；前端的组件化，方便UI组件的重用；

定义全局组件

  • Vue.extend 配合 Vue.component

  • Vue.component

  • 将模板字符串

    1. 组件可以有自己的 data 数据
    2. 组件的 data 和 实例的 data 有点不一样,实例中的 data 可以为一个对象,但是 组件中的 data 必须是一个方法
    3. 组件中的 data 除了必须为一个方法之外,这个方法内部,还必须返回一个对象才行;
    4. 组件中 的data 数据,使用方式,和实例中的 data 使用方式完全一样!!!

父子组件之间的传值

  • 父组件向子组件传值

    一定要使用 props 属性来定义父组件传递过来的数据
    v-bind将数据传递到子组件中

  • 子组件向父组件传值(父组件将方法传给子组件)
    • 原理：父组件将方法的引用，传递到子组件内部，子组件在内部调用父组件传递过来的方法，同时把要发送给父组件的数据，在调用方法的时候当作参数传递进去；
    • 子组件内部通过 this.$emit(‘方法名’, 要传递的数据) 方式，来调用父组件中的方法，同时把数据传递给父组件使用

Vue指令

###Vue的指令

v-cloak插值表达式

解决数据为请求成功之前的闪烁问题

v-text

默认没有闪烁问题，不会展示标签内容

v-bind:

绑定属性 ;简写 : ;只能实现数据的单向绑定 M –> V

v-on

绑定事件 简写 @

v-model

只能用在表单中 input、select、checkbox、textarea ; 实现数据的双向绑定 M –> V V –> M

<input type="text" style="width:100%;" v-model="msg">

v-for

在使用此指令时，必须有 :key属性，代表唯一身份,key的值是num/string，

  • 1. 循环普通数组
    v-for=”(item,i) in list” :key=”i”

  • 2. 循环对象数组
    v-for=”item in list” :key=”item.id”

  • 3. 循环对象
    v-for=”(val,key) in list” :key=”i”

  • 4. 迭代数字
    v-for=”count in 10”

    • in 后面可跟 普通数组，对象数组，对象，数字
    • 循环时数字从1开始

      v-if 和 v-show

      显示后隐藏元素

v-if

  • v-if=”flag = true” —> 显示
    每次点击会删除或重新创建元素

v-show

  • v-show=”flag = false” —> 隐藏
    每次点击会显示或隐藏元素
    当频繁切换使用时用v-show

vue自定义指令

vue.directive(指令名称,对象{bind:function(el){},inserted:function(el){},updata:function(){}},,,)

  • bind : 当元素和指令关联上 调用 —> 执行一次
  • inserted : 当使用指令的元素插入页面时 调用 —> 执行一次
  • updata : 当页面更新时 调用 —> 执行 最少一次，最多无数次

事件修饰符

  • .stop 阻止冒泡 @click.stop=”click”
  • .prevent 阻止默认行为 @click.prevent=”click”
  • .capture 捕获事件，从里到外
  • .self 只触发自己的事件，只会阻止自己的冒泡行为，对别人没有影响
  • .once 事件只触发一次

Vue

不再操作DOM，Vue.js是一套构建用户界面的框架

MVC(Model View Controller)

M:数据层
V:视图层
C:控制层(业务逻辑)

MVVM

  • 前端视图层的分层开发思想，把每个页面分开;
  • 提供数据的双向绑定
  • VM是核心部分，处理数据双向绑定

M: 保存每个页面单独的数据 –> VM将数据存储在M中
VM: 调度者，分割M和V –> VM从M中获取数据；发送给V
V: 每个页面中的HTML结构 –> 将数据存储在VM中

###Vue的使用

在全局下载 npm i Vue Vue-cli -g

用脚手架创建项目

  • vue create “项目名称”
  • cd 到项目根目录下
  • vue run server 运行

生命周期函数

创建

beforeCreate(){}
data和methods未被初始化
8个周期函数创建完成后执行

created(){}
data和methods都已经初始化
如果有页面加载完成就要执行的函数就在此执行

beforeMount(){}
表示模板在内存中编译成功，但未渲染到页面上

mounted(){}
表示内存中的模板已经挂载到页面中，可以操作DOM

运行

当data数据改变时才会执行，最少执行0次

beforeUpdate(){}
数据改变，页面没有改变

updated(){}
页面数据和data数据保持一致，页面上是最新的数据

销毁

beforeDestroy(){}
当执行此函数，Vue实例就已经从运行到销毁阶段
data、methods、过滤器、指令 处于可用状态，还没有真正被销毁

destroyed(){}
组件已经完全被销毁
data、methods、过滤器、指令 不可用

对象

封装了属性和方法，对象是无序列表

面向对象 OOP

Object Oriented Programming 基本概念 一个对象用数据值来描述它的状态；对象及其操作就是对象的行为。对象实现了数据和操作的结合，使数据和操作封装于对象的统一体中 特性

  • 对象唯一性 : 对象都有自身唯一标识
  • 抽象性 : 具有一致的数据结构(属性)和行为(操作)的对象抽象
  • 继承性 : 子类自动共享父类的数据结构和方法机制
  • 多态性 : 相同的操作或函数、过程可作用于多种类型的对象上并获得不同的结果
  • 封装 : 只能通过对象来访问方法 要素
  • 抽象 : 强调实体的本质、内在的属性
  • 封装性 : 保证软件部件具有优良的模块性
  • 共享性 : 面向对象技术在不同级别上促进了共享
